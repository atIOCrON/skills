# Definitions

- A **material finding** identifies a real bug, standards violation, missing
  in-repo reader update, contract break, performance risk, data-quality
  risk, or verification gap, and excludes nits and advisory comments.
- A **root cause** is the underlying issue one or more findings address,
  identified by the same module/file set or the same `docs/` standard rather
  than by overlapping wording.
